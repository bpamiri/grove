# Log Rotation & Cleanup (`grove gc`) Design

**Goal:** Configurable cleanup of accumulated DB rows, log files, and orphaned worktrees to prevent unbounded growth.

**Problem:** Events, sessions, log files, and worktrees grow indefinitely. No cleanup mechanism exists. Long-running Grove installations accumulate dead data from completed/failed tasks.

## Command Interface

**`grove gc [--logs] [--events] [--sessions] [--worktrees] [--all] [--older-than DURATION] [--force] [--dry-run]`**

| Flag | Purpose |
|------|---------|
| `--logs` | Clean `.log` files from `~/.grove/logs/` |
| `--events` | Clean event rows from DB |
| `--sessions` | Clean session records from DB |
| `--worktrees` | Remove worktrees for terminal tasks |
| `--all` | Shorthand for all four above |
| `--older-than DURATION` | Age threshold (default: `30d`). Supports `7d`, `2w`, `3m` |
| `--force` | Actually delete (default is dry-run) |
| `--dry-run` | Preview only (this is the default, explicit flag for clarity) |

No flags specified (no category selected) → show help text. Don't assume `--all`.

### Examples

```
grove gc --all                    # dry-run: preview everything >30d
grove gc --all --force            # actually clean everything >30d
grove gc --logs --older-than 7d   # preview log files >7 days old
grove gc --logs --older-than 7d --force   # delete them
grove gc --worktrees --force      # clean orphaned worktrees >30d
```

## Safety Invariant

**Only data belonging to tasks in terminal status (done, completed, failed) AND older than the threshold is eligible for cleanup.** Data for running, paused, ready, ingested, or planned tasks is never touched regardless of age.

## Cleanup Logic Per Category

### Events (`--events`)

Delete rows from `events` table where the associated `task_id` has terminal status AND `created_at` is older than threshold.

### Sessions (`--sessions`)

Delete rows from `sessions` table for terminal tasks older than threshold. Also clears `session_id` and `session_summary` fields on the task row (the task row itself stays as historical record).

### Logs (`--logs`)

Scan `~/.grove/logs/*.log` files. Match log filename to task ID (format: `{TASK_ID}-{timestamp}.log`). Delete if task is terminal AND file mtime is older than threshold. Orphaned logs (task ID not found in DB) are also eligible.

### Worktrees (`--worktrees`)

Query tasks with non-empty `worktree_path` that are terminal + older than threshold. Use existing `cleanupWorktree()` from `worktree.ts`. Also report orphaned worktree directories (exist on disk but task is missing).

### Deletion Order

Events → sessions → logs → worktrees. DB rows first (lightweight), disk operations last (slower, more destructive).

## Output Format

### Dry-run (default — grouped detail)

```
Grove GC — Dry Run (older than 30 days)

  Events
    W-001  24 rows  (completed 2026-02-01)
    W-003  18 rows  (failed 2026-01-15)

  Sessions
    W-001  1 session  (completed 2026-02-01)
    W-003  2 sessions (failed 2026-01-15)

  Logs
    W-001-20260201-143022.log  1.2 MB
    W-003-20260115-091544.log  0.8 MB
    W-003-20260116-102233.log  0.3 MB

  Worktrees
    W-003  ~/code/wheels/.grove/worktrees/W-003

  Total: 42 events, 3 sessions, 3 log files (2.3 MB), 1 worktree
  Run with --force to clean.
```

### Execute (`--force` — summary only)

```
Grove GC — Cleaned (older than 30 days)
  Events:     42 rows deleted
  Sessions:   3 rows deleted
  Logs:       3 files removed (2.3 MB)
  Worktrees:  1 removed
```

## Duration Parsing

- `7d` / `14d` / `30d` — days (default unit)
- `2w` — weeks (×7 days)
- `3m` — months (×30 days)
- Default: `30d`
- Invalid input → `ui.die()` with usage hint

## Decisions

- **Configurable categories** — flags to target specific areas, `--all` for everything. Avoids accidentally wiping cost/session data still needed for reports.
- **Status-based + age** — only terminal task data (done/completed/failed) older than threshold. Never touch active tasks. Safety over convenience.
- **Dry-run default** — `grove gc` previews by default, `--force` to execute. Worktree deletion is destructive and deserves extra caution.
- **Detail in dry-run, summary in execute** — when previewing you want to review per-task detail. When executing you already reviewed, so summary suffices.
- **No category = help** — running bare `grove gc` without specifying what to clean shows help, not `--all`. Avoids surprises.
- **Reuse `cleanupWorktree()`** — existing function handles git worktree remove + prune. No new worktree logic needed.
- **Orphaned log/worktree detection** — log files and worktree dirs that don't match a DB task are also eligible. These are leftovers from manual deletes.
