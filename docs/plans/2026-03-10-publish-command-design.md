# Design: `grove publish` — Auto Push + PR Creation

## Problem

Workers complete tasks locally (commits on a branch in a worktree) but nothing pushes the branch or creates a PR. The `pr_url`, `pr_number` DB fields and `PrCreated` event type exist but are never populated. Users must manually push and create PRs.

## Decision

Create a `grove publish` command that pushes the branch and creates a draft PR with an AI-generated description. Call it automatically after worker success in `work.ts` and `resume.ts`. On failure, leave status at `done` — work is safe locally and publish can be retried.

## Design

### New file: `src/commands/publish.ts`

#### `publishTask(taskId: string, db: Database): Promise<boolean>`

Exported for use by work.ts/resume.ts. Returns true on success, false on failure.

Steps:
1. Validate task has `branch`, `worktree_path` (exists on disk), and `repo`
2. Resolve `ghRepo` string from `configRepoDetail()`
3. Check for existing PR on this branch via `gh pr list --head {branch} --repo {ghRepo} --json number,url --limit 1`
   - If PR exists: store its info, move to `review`, return true (skip creation)
4. `git push -u origin {branch}` from the worktree cwd
   - On failure: log warning, return false
5. Generate PR body:
   - Get diff: `git diff main...HEAD` in worktree (cap at 50K chars)
   - Spawn: `claude -p "{prompt}" --max-turns 1 --output-format text` with diff as context in the prompt
   - Prompt: "Write a concise PR description for these changes. Include ## Summary (2-3 bullets) and ## Test plan (checklist). Here is the diff:\n\n{diff}"
   - Fallback: if Claude fails, use `session_summary` from the task
   - Append footer: `\n\n---\nGenerated by [Grove](https://github.com/bpamiri/grove) | Task: {taskId}`
6. `gh pr create --draft --title "grove({taskId}): {title}" --body "{body}" --repo {ghRepo} --head {branch}`
   - Parse JSON output for `number` and `url`
   - On failure: log warning, return false (branch is pushed, PR can be created manually)
7. Store `pr_url` and `pr_number` in tasks DB
8. Log `PrCreated` event
9. Move status from `done` to `review`
10. Return true

#### Command entry point: `grove publish [TASK_ID]`

- If TASK_ID provided: publish that task
- If no TASK_ID: find all tasks with status `done` that have a branch, offer to publish
- Validates task status is `done` (or `review` for re-run)
- Calls `publishTask()`

### Changes to `src/lib/github.ts`

Add helper:

```typescript
export function ghPrCreate(repo: string, opts: {
  title: string;
  body: string;
  head: string;
  draft?: boolean;
}): { number: number; url: string }
```

Uses `gh pr create --json number,url` for structured output.

### Changes to `src/commands/work.ts`

After worker exits successfully (exit code 0) and status is set to `done`, add:

```typescript
// Auto-publish: push branch + create draft PR
const published = await publishTask(taskId, db);
if (published) {
  ui.success(`PR created for ${taskId}`);
} else {
  ui.warn(`Auto-publish failed for ${taskId}. Run 'grove publish ${taskId}' to retry.`);
}
```

Same in the background task completion handler.

### Changes to `src/commands/resume.ts`

Same auto-publish call after successful completion.

### Register command in `src/index.ts`

Add `publish` to the lazy-loaded command map.

## Status Flow

```
Before:  worker done → status=done → [gap] → grove review (finds nothing)
After:   worker done → auto-publish → status=review → grove review → merge → grove done
```

## Error Handling

| Failure | Behavior |
|---------|----------|
| Task missing branch/worktree/repo | Log error, return false, stay at `done` |
| PR already exists for branch | Store existing PR info, move to `review`, return true |
| `git push` fails | Log warning, return false, stay at `done` |
| Claude summary fails | Fall back to session_summary as PR body |
| `gh pr create` fails | Log warning, return false, stay at `done` (branch already pushed) |
| Network timeout | Same as push/PR failure — stay at `done` |

## AI Summary Generation

- Input: `git diff main...HEAD` (capped at 50K chars)
- Model: whatever `claude` CLI defaults to
- Flags: `--max-turns 1 --output-format text` (no tool use, plain text output)
- Prompt: structured request for ## Summary + ## Test plan
- Cost: ~$0.01-0.05 per PR (small diff = small input)
- Fallback: session_summary from task DB if Claude call fails

## Unresolved Questions

- Should `--max-turns 1` be sufficient to prevent tool use, or do we also need `--no-tools`? (Need to check Claude CLI docs)
- Should the diff cap be 50K chars or should we use `--stat` + truncated diff for very large changes?
