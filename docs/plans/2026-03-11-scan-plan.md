# Auto-Discover Work (`grove scan`) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a `grove scan` command that auto-discovers actionable work from configured repos via code markers (TODO/FIXME), toolchain signals (outdated deps, lint), and AI-powered deep analysis, with three output modes and deterministic dedup.

**Architecture:** New lib file `src/lib/scanner.ts` with pure scanning functions for three tiers (markers, signals, deep). New command file `src/commands/scan.ts` that orchestrates scanning across repos and handles output modes (dry-run default, apply, interactive). Each tier produces `Finding[]` objects with deterministic `sourceRef` for dedup against existing tasks. Registered in `src/index.ts` and help listing.

**Tech Stack:** TypeScript, bun:test, Node.js fs (file walking), Bun.spawnSync (toolchain commands + Claude CLI)

Imports: `readdirSync`, `readFileSync`, `existsSync` from `node:fs`; `join`, `relative`, `extname` from `node:path`

**`Finding` interface (exported):**

```
export interface Finding {
  repo: string;
  tier: "marker" | "signal" | "deep";
  type: string;       // "TODO", "FIXME", "HACK", "XXX", "DEPRECATED", "outdep", "lint", "smells", "tests", "security"
  file: string;       // relative path from repo root
  line: number | null;
  title: string;
  description: string;
  sourceRef: string;   // deterministic dedup key
  priority: number;    // 50 default, 30 for security
}
```

**Constants:**

- `MARKER_RE = /\b(TODO|FIXME|HACK|XXX|DEPRECATED)\b[:\s]*(.*)/i`
- `IGNORE_DIRS` — Set containing: `node_modules`, `.git`, `vendor`, `dist`, `build`, `.next`, `__pycache__`, `.venv`, `venv`, `.tox`, `target`, `coverage`
- `BINARY_EXTS` — Set containing: `.png`, `.jpg`, `.jpeg`, `.gif`, `.ico`, `.webp`, `.svg`, `.woff`, `.woff2`, `.ttf`, `.eot`, `.zip`, `.tar`, `.gz`, `.bz2`, `.pdf`, `.bin`, `.exe`, `.so`, `.dylib`, `.dll`, `.mp3`, `.mp4`, `.wav`, `.mov`, `.sqlite`, `.db`

**`generateSourceRef(...parts: string[]): string`** (exported):
- Returns `parts.join(":")`

**`walkFiles(dir: string): string[]`** (internal helper, not exported):
- Recursive walk using `readdirSync(dir, { withFileTypes: true })`
- Skip entries whose name is in `IGNORE_DIRS`
- Skip files whose `extname()` is in `BINARY_EXTS`
- Return absolute paths

**`scanMarkers(repoPath: string, repoName: string, limit = 50): Finding[]`** (exported):
- Call `walkFiles(repoPath)` to get file list
- For each file, `readFileSync` (try/catch, skip on error), split lines
- Match each line against `MARKER_RE`
- On match, extract marker type (uppercased) and trailing text
- Build Finding with: `tier: "marker"`, `type: marker`, `file: relative(repoPath, filePath)`, `line: i + 1`, `title`, `description`, `sourceRef: generateSourceRef("scan", repoName, relPath, String(lineNum), marker)`, `priority: 50`
- Break at limit
- Return findings array

**`Toolchain` interface (exported):**

```
export interface Toolchain {
  runtime: "bun" | "node" | "python" | null;
  hasLint: boolean;
  lintTool: string | null;
}
```

**`detectToolchain(repoPath: string): Toolchain`** (exported):
- Check `existsSync` for `bunfig.toml` or `bun.lockb` in repoPath -> `runtime: "bun"`
- Check `package.json` without bun markers -> `runtime: "node"`
- Check `pyproject.toml` or `setup.py` -> `runtime: "python"`
- Check for `.eslintrc*` or eslint in package.json -> `hasLint: true, lintTool: "eslint"`
- Check for `ruff.toml` or `[tool.ruff]` in pyproject.toml -> `hasLint: true, lintTool: "ruff"`
- Default: `{ runtime: null, hasLint: false, lintTool: null }`

**`parseNpmOutdated(jsonStr: string): { pkg: string; current: string; latest: string }[]`** (exported for testing):
- Parse JSON object from `npm outdated --json` output
- For each package, compare major version of `current` vs `latest`
- Only include if major version differs (parseInt first segment before ".")
- Return array of `{ pkg, current, latest }`

**`scanSignals(repoPath: string, repoName: string, limit = 50): Finding[]`** (exported):
- `detectToolchain(repoPath)`
- If runtime is bun/node: run `npm outdated --json` via `Bun.spawnSync` with 30s timeout
  - Parse stdout with `parseNpmOutdated`, create Finding per result
  - Each finding: `tier: "signal"`, `type: "outdep"`, `title: "Outdated: {pkg} {current} -> {latest}"`
  - `sourceRef: generateSourceRef("scan", repoName, "signal", "outdep", pkg)`
- If runtime is python: run `pip list --outdated --format=json` similarly
- If hasLint: run lint command, parse warnings/errors
- Each finding: `priority: 50`
- Wrap all Bun.spawnSync calls in try/catch — failures are non-fatal, return `[]`
- Cap at limit

**Step 3: Implement deep scanning**

**`DEEP_PROMPTS` constant (exported for testing):**

Map of category name to instruction string:
- `smells`: "Identify: dead code, overly complex functions (>50 lines), missing error handling, inconsistent patterns, code duplication."
- `tests`: "Identify: modules with no test coverage, functions with untested edge cases, missing integration tests."
- `security`: "Identify: potential vulnerabilities — SQL injection, XSS, hardcoded secrets, unsafe deserialization, command injection, OWASP top 10."

**`collectSourceFiles(repoPath: string): { relPath: string; content: string }[]`:**
- Reuse `walkFiles()` to get file paths
- Read each file (try/catch, skip on error)
- Skip files > 100KB (likely generated)
- Return array of `{ relPath: relative(repoPath, filePath), content }`

**`buildDeepPrompt(files: { relPath: string; content: string }[], categories: string[]): string`** (exported for testing):
- Build prompt string with:
  - Instructions to analyze source files and return a JSON array
  - Each entry schema: `{ file, line, category, title, description }`
  - Category instructions (only for requested categories)
  - File contents with `--- FILE: {path} ---` separators
  - Footer: "Return ONLY a JSON array. No other text."
- Only include categories the user requested from DEEP_PROMPTS

**`parseDeepResponse(response: string, repoName: string): Finding[]`** (exported for testing):
- Try to parse response as JSON array
- If not an array, try to extract JSON array from text via regex
- For each entry in parsed array, map to Finding:
  - `tier: "deep"`, `type: entry.category`
  - `priority: entry.category === "security" ? 30 : 50`
  - `sourceRef: generateSourceRef("scan", repoName, "deep", entry.category, entry.file, simpleHash(entry.title))`
- `simpleHash(str)`: first 8 chars of hex from simple character-code sum

**`scanDeep(repoPath: string, repoName: string, categories: string[], limit = 50): Finding[]`** (exported):
- Collect source files via `collectSourceFiles(repoPath)`
- Calculate estimated tokens (total chars / 4)
- Chunk files into ~50KB batches (by cumulative content size)
- For each chunk:
  - Build prompt via `buildDeepPrompt(chunk, categories)`
  - Spawn: `Bun.spawnSync(["claude", "-p", prompt, "--output-format", "json"], { cwd: repoPath, timeout: 120_000 })`
  - Parse stdout as JSON, extract `.result` field
  - Call `parseDeepResponse(result, repoName)`
- Aggregate findings, cap at limit
- Return findings
