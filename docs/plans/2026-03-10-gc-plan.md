# Log Rotation & Cleanup (`grove gc`) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a `grove gc` command that cleans up old DB rows, log files, and orphaned worktrees for terminal tasks, with configurable categories, age threshold, and dry-run-by-default safety.

**Architecture:** New command file `src/commands/gc.ts` with pure helper functions for duration parsing, candidate collection (per category), and execution. Each category (events, sessions, logs, worktrees) has its own collector function that returns a typed results object. Dry-run renders detailed per-task breakdown; `--force` executes and shows summary. Registered in `src/index.ts` and help listing.

**Tech Stack:** TypeScript, bun:sqlite (via existing Database class), Node.js fs (for log file operations), bun:test

### Task 2: Implement the gc command

**Files:**
- Create: `src/commands/gc.ts`

**Step 1: Create the gc command**

The command exports `gcCommand: Command` and `parseDuration` (exported for testing).

**Constants:**
- `TERMINAL_STATUSES = new Set(["done", "completed", "failed"])` — status gate
- `DEFAULT_DURATION = "30d"` — default age threshold

**`parseDuration(input: string): Date | null`:**
- Regex match: `/^(\d+)(d|w|m)$/`
- d = days, w = weeks (x7), m = months (x30)
- Returns `new Date(Date.now() - days * 86400_000)` or null if invalid

**Candidate types:**
- `EventCandidate`: `{ taskId, taskStatus, taskUpdatedAt, count }`
- `SessionCandidate`: `{ taskId, taskStatus, taskUpdatedAt, count }`
- `LogCandidate`: `{ filePath, fileName, taskId, sizeBytes, orphaned }`
- `WorktreeCandidate`: `{ taskId, worktreePath, taskStatus }`
- `GcCandidates`: `{ events[], sessions[], logs[], worktrees[] }`

**`collectEvents(db, cutoff)`:**
- SQL JOIN tasks+events, filter by terminal status AND `updated_at < cutoff`
- GROUP BY task_id to get count per task

**`collectSessions(db, cutoff)`:**
- Same pattern as events but JOIN tasks+sessions

**`collectLogs(db, cutoff, logDir)`:**
- Scan `logDir/*.log` files
- Parse task ID from filename pattern `{TASK_ID}-{YYYYMMDD}-{HHMMSS}.log`
- Check file mtime against cutoff
- Check task status (terminal = eligible, active = skip)
- Missing task in DB = orphaned = eligible

**`collectWorktrees(db, cutoff)`:**
- Query tasks with terminal status, older than cutoff, non-empty worktree_path
- Filter to those where directory still exists on disk

**`formatBytes(bytes)`:** Human-readable size (B, KB, MB)

**`renderDryRun(candidates, durationLabel)`:**
- Grouped detail output: Events, Sessions, Logs, Worktrees sections
- Per-task line with count/size details
- Total summary line
- "Run with --force to clean." footer

**`executeGc(db, candidates, durationLabel)`:**
- Delete events: `DELETE FROM events WHERE task_id = ?` per candidate
- Delete sessions: `DELETE FROM sessions WHERE task_id = ?` per candidate
  - Also: `UPDATE tasks SET session_id = NULL, session_summary = NULL WHERE id = ?`
- Delete logs: `unlinkSync(filePath)` per candidate
- Remove worktrees: `cleanupWorktree(taskId, db)` per candidate (reuse existing)
- Summary output with counts

**`gcCommand.run(args)`:**
1. Help check (-h, --help)
2. Parse flags: --events, --sessions, --logs, --worktrees, --all, --force, --dry-run, --older-than
3. No category selected → show help
4. Validate duration via `parseDuration()`
5. Collect candidates per enabled category
6. If --force → `executeGc()`, else → `renderDryRun()`

**`help()`:** Usage text with all flags, categories, safety note, examples
